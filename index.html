Glock Bird

Previous Next

About

Glock Bird is a web-game on Itch.io that takes familiar mechanics of a bird flying between obstacles while adding its own twist by giving that bird a gun. The game features cutom-made sprites and music with inspiration from retro arcade games.

The Process

Glock Bird is my first project in the Unity Engine and it really challenged me to learn everything I could about Unity’s interface while also trying to make a fun experience. In my pursuit I learned project management, programming, and game design skills that will carry me a long way. Also, in adjacent to this game’s development I was also going through Unity’s “Create with Code” course on their website to make sure I was not missing any important concepts with Unity’s interface. I used resources such as Miro Board, Notion, and Google Docs to design the game. I also designed the music and sprites for the game. To do this I used FamiStudio, Pixilart, and Krita.

Below, you can see some illustrations and documentation created for organization:


For this project, I used Unity’s Plastic SCM to commit my changes to a branch so no precious progress would be lost when I inevitably made some mistakes that I’d wanna revert. This game has challenged me to think critically about game architecture because I constantly found myself going back trying to find different ways to optimize my code.

Cursed Dungeon

Previous Next

About

Cursed Dungeon is a retro-style arcade game where the player is trapped in an endless dungeon where every room looks the same! The player has to complete a timing challenge at every door in order to progress to the next room, and the challenges become more and more frantic as the player progresses. Everything from the art, the coding, and the design was done by me in creating this game. Also, The game was designed for the Boss Rush Game Jam 2025 on Itch, the theme being "spin."

Cursed Dungeon was made using Unity Engine and the art was made using Resprite for IOS. GitHub was used for version control and the code is open-source for anyone to view.

Designing the Game

I joined the game jam around 2 weeks before it was due, so I had to quickly come up with an idea and begin implementation as soon as possible. I drew up some project sketches and diagrams, which can be found posted on my Itch page. I knew I wanted to do something retro and dungeon-like because I am a huge fan of dungeon-crawler games. I settled on a quick idea I could implement with an eyeball spinning inside of a stone mask. I didn't wanna bog myself down with controls so I decided to only add one control, and that is to simply stop the eyeball when it lands on the lock sigil.

Challenges and Takeaways

The first week of development was spent mostly on implementation. I used component-oriented design whenever possible to try to keep my code modular and scalable. A challenge I faced with this phase was sacrificing some intuitive design-choices for time. Time became my biggest hinderance when it came to this project because of classes. Still, my code turned out to be passably scalable by the time the game released.

The second week of the game's development was entirely spent on looks and polishing. I had to rethink how I approached the scripting and animations several times to work with new things I wanted to add. I learned a ton about making scripting that works with game animations and sounds through this process.

Overall this project taught me a lot about game flow and intuitive game design. If you would like to try the game yourself, click the link below!
